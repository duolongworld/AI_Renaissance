# 宏观分析 Skill 输入数据规范（Input Data Specification）

> **文档版本**：V1.0  
> **创建日期**：2026-05-12  
> **适用范围**：宏观分析 Skill（6层流水线）完整流程  
> **数据来源**：需求文档（requirement.md）+ 框架手册 V3.0  

---

## 1. 文档说明

本文档定义宏观分析 Skill 6层流水线（Layer 0 → Layer 5）所需的**原始输入数据**，用于指导数据组同事采集和提供数据。

**本文档不包含**：
- 由原始数据二次处理计算的衍生指标（如 CAI、FCI、通胀得分、预期差等）
- 这些衍生指标的计算逻辑请参考《需求文档》各 Layer 详细说明

**文档包含**：
- **数据定义名**：指标的标准名称与口径
- **数据来源**：官方/第三方数据源
- **采集频率**：日频/周频/月频/季频
- **用途说明**：该数据在哪个 Layer 使用，用于什么计算

**数据分类**：
1. **必填输入**：Skill 运行必需的核心数据
2. **可选输入**：增强分析精度或处理特殊场景的辅助数据

## 5. 数据口径强制规范

（参考框架手册 3.3 节）

| 指标类别 | 口径规范 | 说明 |
|---|---|---|
| **PMI类指标** | 全球/国内统一采用官方季调后口径 | 避免季节性扰动 |
| **同比类指标** | 除特殊说明外，均采用"季调后3个月移动平均"口径 | 平滑月度波动 |
| **利率/汇率/波动率类指标** | 均采用"日度均值→月度滚动平均"口径 | 避免单日异常值 |
| **剪刀差/占比类指标** | 均采用"当月原始数据计算"，不做移动平均 | 保持原始关联性 |
| **z-score标准化** | 均采用"5年滚动z-score" | 保证跨期可比性 |

## 6. 数据缺失处理规则

（参考需求文档 2.3 节）

| 缺失情形 | 处理规则 | 输出影响 |
|---|---|---|
| 单个指标临时缺失（如PMI尚未发布） | 使用过去3个月均值填充，标注 `uncertainties` | 信号置信度下调0.1 |
| 多个指标同时缺失（>3个） | 暂停该层分析，输出 `neutral` 信号，`needs_human_review = true` | 仅输出观测性描述，不输出方向性信号 |
| 数据源完全中断（如Wind/Bloomberg故障） | 启动备用数据源（如改用NBS/央行官网手动录入），标注 `evidence.source_type = "manual_backup"` | 信号置信度下调0.2 |
| 低频数据尚未发布（如季度GDP） | 使用高频指标插值估算，标注 `uncertainties` | 仅用于趋势判断，不用于精确点位 |

## 8. 数据质量检查清单

### 9.1 数据完整性检查

- [ ] 必填输入指标是否全部获取成功
- [ ] 数据日期是否为最新（符合采集频率要求）
- [ ] 数据数值是否在合理范围内（无异常值）
- [ ] 数据口径是否符合规范（季调/移动平均/原始值）

### 9.2 数据一致性检查

- [ ] 同一指标的不同来源是否一致（如Wind vs Bloomberg）
- [ ] 相关指标之间是否逻辑一致（如PMI > 50 对应工业增加值同比 > 0）
- [ ] 跨境指标是否汇率调整一致（如中美利差计算使用同一天汇率）

### 9.3 数据时效性检查

- [ ] 日频数据是否在T+1日内获取
- [ ] 周频数据是否在周末后第一个工作日获取
- [ ] 月频数据是否在发布后2个工作日内获取
- [ ] 季频数据是否在发布后5个工作日内获取
