一、什么是宏观分析

本手册所指宏观分析，是以经济增长、流动性、通胀、政策、全球传导为核心变量，判断周期位置与边际变化。

1.1 宏观分析的底层公理（五大公理）

  ---------------------------------- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  底层公理                           核心本质                                                                                                                                                                                                                                                           对我们的约束                                                                                                                                                                                                             对应执行动作

  **公理 1：资产定价双因子公理**     权益、债券类核心资产价格 = 未来现金流 / (1 + 折现率)\^n；分子 = 企业盈利（由经济增长周期决定），分母 = 无风险利率 + 风险溢价（由流动性 + 市场情绪决定）；黄金、大宗商品等另类资产需匹配专属定价逻辑                                                                我们的框架只分「增长周期」（分子端）和「流动性周期」（分母端），仅跟踪与这两个因子强相关的有效指标，另类资产单独补充对应指标；**AI 科技是增长周期的核心驱动力**，高技术制造业 PMI 与 AI 资本开支是分子端的关键代理变量   将核心指标划分为「国内基本面轮（分子端）」和「全球流动性轮（分母端）」，其他所有因素最终仅能通过这两个变量影响核心资产价格

  **公理 2：周期运行规律公理**       经济和流动性的运行不是随机波动，而是由信贷、库存、产能、债务的循环驱动，形成典型的短 / 中 / 长周期规律；受外生冲击（地缘、疫情、超预期政策）周期可能出现阶段跳跃，拐点有明确的领先信号                                                                             我们需要通过分层的量化指标，精准判断当前所处的周期阶段，区分趋势性变动与短期波动                                                                                                                                         执行「长周期→中周期→短周期」的三级定位规则，不用短期波动否定长期趋势，外生冲击下需启动高频指标跟踪校验

  **公理 3：全球传导公理**           美元本位体系下，美联储是全球流动性总闸门，其政策决定全球美元流动性总量，进而影响全球风险资产估值中枢、跨境资本流向；常规场景下全球流动性周期优先级高于国内流动性周期，国内强政策对冲场景可调整权重                                                                 分析必须遵循「先全球、后国内」的核心路径；全球流动性持续收紧时，国内宽松难以催生全面牛市，仅存在结构性机会                                                                                                               信号优先级常规执行「全球流动性＞国内货币＞国内增长＞美国基本面＞全球风险」，设置国内强政策对冲的例外调整规则

  **公理 4：预期差定价与驱动公理**   资产价格已提前反映市场一致预期，只有实际数据 / 政策超出 / 不及市场预期时才会引发重新定价------这是**定价机制**；而预期差的大小和方向直接决定超额收益的空间，预期差越大定价修正空间越大------这是**Alpha 驱动机制**。仅凭数据方向交易本质上是在交易共识而非 Alpha   我们不仅要看数据绝对值，更要对比数据与市场一致预期的差值；必须同时掌握基本面实际状态和市场已定价状态，二者的差才是可交易的信号                                                                                           建立 Layer 3（市场定价提取）和 Layer 4（预期差信号引擎），系统化提取三类预期差信号（数据意外、基本面 vs 市场定价、跨国预期差）；用花旗经济意外指数做交叉验证，所有指标口径必须与市场一致预期统计口径对齐

  **公理 5：双市场联动公理**         A 股与美股在全球流动性周期上高度同步，但在盈利周期（分子端）上存在结构性分化。中美经济周期错位为跨市场结构性配置提供机会窗口                                                                                                                                       不能以单一市场周期结论套用至另一市场；需专门建立「中美周期对比模块」，识别分化与收敛节点                                                                                                                                 当中美周期方向一致时，可输出两市场共振信号；严重分化时，仅输出结构性配置信号，不做单一全面方向判断；美国基本面独立模块纳入 Layer 0 并行追踪

1.2 宏观分析的核心逻辑

基于六大底层公理，搭建「五轮驱动、三级周期、7 层流水线」的标准化分析体系：

1.  **五轮驱动：** 以「全球流动性轮（共同分母端）」「美国基本面轮（美股分子端）」「中国基本面轮（A 股分子端）」「全球风险轮（共同风险层）」「AI 科技结构轮（行业配置层，聚焦科技主线）」为五大核心，覆盖 A 股与美股双市场的核心定价变量；

2.  **三级周期：** 以「长周期定方向、中周期定中枢、短周期定节奏」的三级定位体系，精准判断周期所处阶段；

3.  **7 层流水线：** Layer 0（双经济体追踪）→ Layer 1（状态识别 CAI/FCI）→ Layer 2（周期定位 4 象限+政策调节）→ Layer 2.5（枢纽变量分析）→ Layer 3（市场定价提取）→ Layer 4（预期差信号引擎）→ **Layer 4.5（反身性与元认知层）** → Layer 5（资产配置 Beta+Alpha）

------------------------------------------------------------------------

1.3 我们解决的 3 个核心问题

1.  **顶层约束问题：** 全球核心经济体处在什么周期阶段？全球流动性与风险环境的顶层约束是什么？对 A 股的传导路径与影响力度如何？

2.  **周期定位问题：** 在全球约束下，国内增长周期（分子端）与流动性周期（分母端）分别处于长 / 中 / 短周期的哪个阶段？未来边际变化的领先信号与预期差是什么？

3.  **定价决策问题：** 基于全球 + 国内周期组合，对应 A 股大类资产应输出什么方向、多强强度、多久有效期的择时与配置信号？**AI 科技主线的周期定位与配置权重如何确定？**

三、宏观分析的核心指标体系

3.1 指标选取核心原则

1.  **领先性优先原则：** 优先选取领先于经济周期、领先于资产价格的指标，滞后指标仅可用于交叉验证，不得单独用于拐点预判与实时决策；

2.  **可量化原则：** 指标需有明确的数值、可标准化统计，禁止使用无量化标准的定性描述类指标；

3.  **口径统一原则：** 所有指标均固定统计口径、频率、来源，不随意切换季调 / 非季调、同比 / 环比口径，且必须与市场一致预期的统计口径对齐；

4.  **强相关性原则：** 指标需与宏观周期、A 股大类资产价格高度相关，剔除无关、弱相关指标，避免冗余，同一信号仅保留 1 个核心指标 + 1 个备用指标；**AI 科技主线优先**------在同等信号质量下，优先保留与科技产业链、高技术制造业、AI 资本开支强相关的指标；

5.  **可获取原则：** 指标数据需公开可获取、及时更新，确保日常分析可落地，避免使用难以获取的小众指标；

6.  **频率分层原则：** 按日 / 周 / 月 / 季频分层设置指标，高频指标跟踪边际变化，低频指标确认周期趋势，解决滞后性问题；

7.  **AI 可执行原则：** 所有指标规则可代码化、无模糊空间，支持 Skill 自动抓取、清洗、计算；

8.  **合规性原则：** 所有数据来源均为官方合规渠道，无违规数据获取。

3.3 指标口径强制规范

1.  **PMI 类指标：** 全球 / 国内统一采用官方季调后口径，与市场一致预期统计口径完全对齐；无季调数据仅作为补充参考，不纳入核心决策；

2.  **同比类指标：** 除特殊说明外，均采用「季调后 3 个月移动平均」口径，规避基数效应和单月异常波动；拐点判断采用「当月同比值较上月同比值的环比变动」，统一时间维度，避免同比 / 环比混用；

3.  **利率 / 汇率 / 波动率类指标：** 均采用「日度均值→月度滚动平均」口径，平滑日内异常波动；趋势判断以「连续 1 个月变动方向」为标准，不根据单日波动调整结论；

4.  **剪刀差 / 占比类指标：** 均采用「当月原始数据计算」，不做移动平均，确保信号时效性；

5.  **数据修正规则：** 所有指标数据以官方最终修正版为准，仅修正未来的周期判断与信号输出，不追溯过往已发布的结论；

6.  **指标替代规则：** 核心指标缺失时，仅可使用同类型、同口径、同领先性的替代指标，且必须在数据清单中明确标注替代原因、替代周期，禁止使用无关指标替代；

7.  **异常值处理规则：** 单月 / 单日指标出现异常跳升 / 跳落，不直接判定趋势反转，需等待连续 2 个月的同向数据确认，方可调整周期判断；

8.  **花旗经济意外指数量化阈值：** 指数数值以标准差（σ）为单位，量化判定规则如下：

    - 指数 \> **+0.5σ**：认定为**正**（数据超预期，+1 分）

    - 指数 \< **-0.5σ**：认定为**负**（数据不及预期，-1 分）

    - 之间（-0.5σ \~ +0.5σ）：认定为**中性**，不参与有效性加减分

9.  **核心 CPI 温和区间量化：** 统一明确为 **0.5% \~ 2.5%**；低于 0.5% 为通缩压力区，高于 2.5% 为通胀压力区，均触发货币政策边际调整预判；

10. **「国内强政策对冲」量化标准：** 需同时满足以下至少 **2 项**，方可触发优先级调整（国内货币/增长模块优先级提升至与全球流动性模块持平）：

    - 财政政策：单次刺激规模 **\> 1.5 万亿人民币**（如特别国债、财政赤字率大幅提升）

    - 货币政策：降准幅度 **\> 50BP**，或降息幅度 \> 20BP（MLF/LPR 同步）

    - 科技/产业政策：**同时出台 2 项以上**重点产业支持政策（如 AI 算力补贴、半导体基金、新能源消纳保障）

11. **「幅度达标 ≥ 0.2 个百分点」口径明确：** 所有「单月变动幅度」均指**季调后数据的环比变动**（即当月季调后值减去上一月季调后值的绝对差值），未经季调的原始同比/环比变动不单独作为判定依据；

12. **北向资金有效信号阈值：** 原规则「连续 2 周净流入/流出」易受短期噪音干扰，调整为：**单月净流入 \> 200 亿认定有效净流入信号，单月净流出 \> 200 亿认定有效净流出信号**，200 亿以内为中性，不单独作为周期判定依据。

3.5 指标有效性回测与迭代规则

1.  **半年度全量回测：** Skill 输出小组每半年对所有核心指标的领先性、相关性、胜率进行全历史回测，剔除胜率低于 **60%** 的弱有效指标，补充高胜率有效指标；

2.  **季度优化校准：** 大周期融合小组每季度根据复盘结果，对指标阈值、口径进行校准，校准方案需经过回测验证、全组评审后方可更新；

3.  **核心指标锁定规则：** 四大模块的 TOP3 核心指标（美联储政策利率、美债收益率、国内 PMI、社融、DR007、10 年期国债收益率）不得随意替换，仅可优化口径与阈值；

4.  **新增指标准入规则：** 新增指标必须满足「3.1 选取原则」，且经过 **10 年以上**历史数据回测，胜率 ≥ **65%**，方可纳入标准化指标清单；

5.  **框架版本触发规则：** 见第六章「框架版本管理机制」。

6.  **假设注册-追踪-验证机制：** 面对历史上真正带来巨大超额收益的机会往往来自\"新逻辑\"而非旧指标优化的事实，框架设立\"观察池\"通道，允许未满足回测门槛的新逻辑/新指标进入追踪体系：

    - **假设注册阶段：** 新逻辑/新指标进入\"观察池\"，标注逻辑路径、预期信号方向、验证条件，**不参与任何权重计算**（资金仓位 0%）

    - **信号追踪阶段：** 追踪该假设在后续 6-12 个月的实际表现（方向命中率、预期差大小），与正式指标平行运行但不交叉影响

    - **验证升级阶段：** 追踪期结束后统计胜率，若 ≥ 60% 且逻辑路径清晰 → 按正式指标准入规则纳入指标体系；若 \< 60% → 从观察池移除并记录失败原因

    - **观察池上限：** 同时追踪的假设不超过 **5 个**，避免观察池泛滥侵蚀框架纪律

    - **禁止事项：** 观察池假设不得以任何形式（包括\"探索性仓位\"）参与实际资金配置

四、7 层流水线标准化执行流程

4.1 核心思维路径

基于六大底层公理，严格遵循「从顶层到底层、从全球到国内、从状态识别到预期差信号、从信号可靠性检验到 Alpha 偏离、从 Beta 基准到 Alpha 偏离」的分层路径：

1.  **先做双经济体追踪：** 同时追踪中美两国 5 维度指标（Layer 0），捕捉跨国信号与交互通道触发状态

2.  **再做状态识别：** 通过 CAI/FCI 合成指标判断中美增长与金融条件的实际状态（Layer 1）

3.  **再做周期定位：** 基于增长×通胀 4 象限模型+政策维度调节+长期债务周期战略偏置，锁定周期位置（Layer 2）

4.  **再做枢纽变量分析：** 通过汇率、大宗商品、商品-汇率-利率三角，判断全球宏观背景（Layer 2.5）

5.  **再做市场定价提取：** 从 5 类市场价格反推市场已定价了什么，与实际状态形成对照（Layer 3）

6.  **再做预期差信号：** 实际状态 vs 市场定价的差值，生成三类预期差信号（Layer 4）

7.  **再做信号可靠性检验：** 反身性压力计 + 逻辑生命周期修正 + 范式稳定性监测，修正信号强度（Layer 4.5）

8.  **最终映射资产配置：** Beta 基准 + Alpha 偏离的双层架构，输出最终权重（Layer 5）

4.2 周期拐点确认规则

4.2.1 核心定位

本章为宏观周期拐点的**量化确认工具**，唯一作用是回答\"周期拐点是否成立\"。4.2 不独立输出周期定性标签，而是为 Layer 2 的四象限定位提供拐点确认依据------当 Layer 2 四象限出现象限切换信号时，必须通过 4.2 的拐点确认标准方可正式判定切换成立，禁止跳过本章直接判定周期。

------------------------------------------------------------------------

4.2.2 标准六阶段固定循环

周期按以下顺序循环运行，受外生冲击可能出现阶段跳跃，但阶段命名与判定标准不变：

**衰退 → 复苏 → 扩张 → 过热 → 放缓 → 衰退**

------------------------------------------------------------------------

4.2.3 拐点统一确认硬性规则

周期拐点的确认，必须同时满足以下三大条件，缺一不可；滞后指标仅做交叉验证，不单独用于拐点判定：

1.  **双指标共振：** 选取对应拐点的核心领先指标（至少 2 个），不得用单一指标判定；

2.  **连续同向：** 核心指标连续 **2 个月**呈现同向变动（回升 / 下行）；

3.  **幅度达标：** 单月变动幅度 ≥ **0.2 个百分点**（季调后数据的环比变动，详见 3.3 第 11 条）。

0.1 设计依据

中国资产的定价同时受国内基本面和美国/全球周期驱动，这是 2018 年以来最重要的结构性变化。仅追踪中国的框架在 2022 年（美联储激进加息）、2024 年（中美利差深度倒挂）等关键时点必然失效。本层设立中国与美国并行追踪结构，作为整个 7 层流水线的输入层。

**核心原则：**

1.  **两国指标按完全对应的维度组织**------每个维度同时跑两国版本，通过差值或比值得到跨国信号。例如\"中美增长差\"\"中美通胀差\"本身就是独立的资产定价变量。

2.  **中国数据的特殊处理**------春节扰动（1-2 月宏观数据合并看待或做春节调整）、数据修订与发布滞后（初值→修订→终值，取发布时间戳）、指标失效替代（如 M2 失去信号意义→以社融存量同比替代）。

3.  **Point-in-time 数据强制要求**------回测必须用当时能看到的版本（vintage data），避免前视偏误。所有历史序列在数据库中按\"发布日期\"而非\"统计期\"排列。

Layer 1：状态识别------CAI/FCI 合成指标

1.1 核心工具：合成指标

单指标噪声大、方向不一致，直接使用容易产生误判。合成指标（synthetic indicator）通过多指标加权聚合，提取共性信号、压制个体噪声。

引入两个核心工具：

- **CAI（Current Activity Indicator，实时活动指数）**：实时 GDP 替代指标，回答\"经济现在好不好\"，捕捉增长方向与力度。

- **FCI（Financial Conditions Index，金融条件指数）**：流动性温度计，回答\"金融环境松还是紧\"，捕捉政策与市场的资金面状态。

离散标签作为兜底（见 1.7 节），连续分数用于预期差信号计算和极端值判断。

2.4 周期一致性校验与联动规则

当全球周期与国内周期出现分歧时，严格遵循以下规则，杜绝逻辑矛盾：

1.  **全球长 / 中周期与国内对应周期分歧**，以全球周期定大方向，国内周期仅调整信号节奏，不改变核心结论；国内强政策对冲场景除外（量化标准见 3.3 第 10 条）；

2.  **全球中周期与国内短周期分歧**，优先参考全球中周期，国内短周期波动不判定为周期拐点；

3.  **全球风险模块触发恐慌状态**，无论全球与国内周期如何，直接下调至观望 / 看空信号；

4.  **中周期必须服从长周期的大方向**，短周期必须服从中周期的中枢定位，禁止反向判定；

5.  **中美周期严重分化时**（公理 5 约束），不得输出全面看涨 / 看空信号，仅可输出结构性配置信号；

6.  **中美周期方向一致时**（公理 5 约束），可输出两市场共振信号，信号强度 +1 分；

7.  **Sahm Rule 触发时**，无论美国基本面模块标签如何，强制标注\"美国衰退预警\"，美股信号下调 1 档。

**输出结果：** 全球 + 美国 + 中国全周期定位结论，无逻辑冲突，与 4.2 阶段定性完全一致

Layer 3：市场定价提取

3.1 设计依据

这是绝大多数传统宏观框架缺失的层次，也是 Alpha 的核心来源。

**核心逻辑：** Alpha 来自\"实际状态 vs 市场定价\"的差------没有这一层就没有差，没有差就没有 Alpha。传统框架止步于\"判断宏观状态\"，但市场已经对宏观状态进行了定价。只有同时掌握基本面实际状态和市场已定价状态，才能识别定价偏差、捕捉超额收益。

**本层职责：** 从市场价格反推市场已定价了什么，与 Layer 1 的实际基本面状态形成对照。Layer 1 回答\"经济实际上怎样\"，Layer 3 回答\"市场认为经济怎样\"------两者的差就是 Layer 4 预期差信号的输入。

本层涵盖原市场预期定价扫描功能，并系统性扩展为五类定价指标的完整提取框架------覆盖利率、估值、汇率、商品、信用五个维度，为 Layer 4 的预期差计算提供结构化输入。

Layer 4：预期差信号引擎

4.1 核心思想：预期差驱动

Surprise Index 体系是本层的起点。传统宏观框架的信号逻辑是\"数据好→做多，数据差→做空\"，但市场价格已经反映了市场对数据的共识预期。仅凭数据方向交易，本质上是在交易共识，而非交易 Alpha。

**Alpha 的真正来源是预期差------实际发生与市场定价之间的偏差。**

本框架将预期差从\"数据 vs 共识\"扩展为三层结构：

  -------------------------------- -------------------------------------- ------------------- ---------------------------
  层次                             对比维度                               信号含义            交易逻辑

  **第一层：数据意外**             数据实际值 vs 市场共识预期             短期方向冲击        事件驱动交易，衰减快

  **第二层：基本面 vs 市场定价**   Layer 1 实际状态 vs Layer 3 市场定价   中期定价修正        核心 Alpha 来源，持续性强

  **第三层：跨国预期差**           中国预期差 vs 美国预期差               相对价值/资本流向   跨市场套利
  -------------------------------- -------------------------------------- ------------------- ---------------------------

本层保留有效性评级的分数框架（4.4 节），将其嵌入完整的预期差信号体系，从\"评级\"升级为\"信号引擎\"。

4.3 信号强度与多元化

信号强度评级

每个预期差信号生成**方向**（正/负）和**强度**（历史百分位）：

- 方向：预期差 \> 0 = 正信号，\< 0 = 负信号

- 强度：取该预期差绝对值的 5 年滚动百分位排名

  - \> 80 分位：强信号

  - 50-80 分位：有效信号

  - \< 50 分位：弱信号

信号来源汇总

  --------------------------------------- ----------------------- ------------------------------------------------------------------------------------
  信号来源                                数量                    示例

  **类型 A：高频意外指数**                2                       中国 CESI、美国 CESI

  **类型 B：基本面 vs 市场定价**          4                       增长/通胀/政策/外部预期差

  **类型 C：跨国预期差**                  6                       中美增长/通胀/政策/流动性/信用/美元-大宗联动

  **类型 D：枢纽变量信号（Layer 2.5）**   4                       铜金比、铁矿石/铜、黄金 vs 实际利率、南华 vs 全球 PMI

  **类型 E：估值与行为**                  2-3                     A 股 ERP 极端值、北向资金极端流出、衍生品市场信号

  **类型 F：元认知修正**                  3                       反身性压力计、范式稳定性、逻辑生命周期（不直接产生新信号，但修正所有现有信号强度）

  **合计**                                **约 23 个**            目标是其中 10-15 个真正不相关；类型 F 为全局修正因子，不参与独立性检验
  --------------------------------------- ----------------------- ------------------------------------------------------------------------------------

投资圣杯原理

**N 个不相关信号叠加，组合夏普 = 单信号夏普 × √N**

- 假设单信号夏普 = 0.3，则 10 个不相关信号组合夏普 = 0.3 × √10 ≈ 0.95

- 关键约束：信号之间必须**低相关**。类型 A/B/C/D/E 的信号来源不同（经济数据、市场定价、跨国差、大宗比值、行为指标），天然具有低相关性；类型 F（元认知修正）为全局修正因子，不产生新信号方向，而是对现有信号强度做乘法修正

- 实际目标：从约 20 个信号中筛选出 10-15 个相关系数 \< 0.3 的信号源叠加

**相关性检验：** 每季度对全部信号做滚动 2 年相关系数矩阵检验，相关系数 \> 0.5 的信号对取其一或降权。

- 信号强度 + 边际 ↘：强度评分 −5 分位（如 82% → 77%，降级为有效信号）

4.6 逻辑生命周期与叙事共识度监控

4.6.1 设计依据

Layer 4 预期差信号引擎计算的是某一时刻\"实际状态 vs 市场定价\"的静态差值。但投资逻辑具有动态生命周期：

**逻辑萌芽**（无人关注）→ **逻辑扩散**（部分人认同）→ **逻辑共识**（广泛接受）→ **逻辑透支**（过度定价）→ **逻辑崩塌**（预期差逆转）

当框架识别到一个预期差信号时，如果不判断该逻辑处于生命周期的哪个阶段，可能出现：基本面数据仍然好看、预期差看起来仍然\"正\"，但逻辑已被市场充分消化，继续交易不再是 Alpha 而是拥挤交易。

**核心原则：** 当预期差信号方向与市场主流共识方向一致时，该预期差已被充分定价，信号强度应自动降低。

Layer 4.5：反身性与元认知层

4.5.1 设计依据

当前框架的 Layer 0-4 是单向传导的：基本面 → CAI/FCI → 四象限 → 市场定价 → 预期差。这个流程隐含一个假设：**市场参与者观察市场，其行为不改变市场本身。**

索罗斯的反身性理论提醒我们：**市场参与者的行为本身也改变市场。** 当越来越多资金采用相似的宏观分析框架和信号时，这些信号本身可能改变市场的运行方式------形成预判的自我实现或自我否定。

本层不是取代现有框架，而是为它装上一层\"元认知\"------回答一个前置问题：**\"当前框架的信号是否仍然可靠？\"** 它是框架的\"观察大脑\"，为\"执行大脑\"（Layer 0-5）保驾护航。

**三层元认知功能：**

4.5.2 反身性压力计

反身性压力计是本层的核心工具，监测\"共识的拥挤程度\"，在大众同向共振时自动降低信号的执行力度。

**四大压力指标：**

  ---------------------- --------------------------------------------------------------------------------------------------- -------------- -------------- ------------------------------------
  指标                   计算方式                                                                                            预警阈值       极端阈值       信号含义

  **信号拥挤度**         采用与框架同方向的策略资金规模 / 市场总容量（以 CFTC 非商业持仓 + ETF 资金流 + 北向资金综合估算）   \> 50%         \> 70%         共识交易占比过高，Alpha 消失

  **信号自我实现指数**   信号发布后价格变动中\"预期驱动\"占比 = （价格变动幅度 − 基本面变化可解释幅度）/ 价格变动幅度        \> 60%         \> 80%         价格变动主要由预期推动而非实际变化

  **跨框架一致性**       主流卖方/买方策略报告与框架信号的吻合度（抽样统计前 10 大卖方 + 前 5 大买方）                       \> 70%         \> 90%         太多人看到同样的信号

  ----------------------- ----------------------- ---------------------------------------------------------------
  综合得分区间            压力等级                对 Layer 5 执行的影响

  0-30                    绿色（低压力）          正常执行，信号强度不修正

  30-50                   黄色（中压力）          Alpha 偏离执行力度降至 **75%**

  50-70                   橙色（高压力）          Alpha 偏离执行力度降至 **50%**，触发人工复核

  70-100                  红色（极高压）          Alpha 偏离执行力度降至 **25%**，强制触发人类裁决接口（4.5.4）
  ----------------------- ----------------------- ---------------------------------------------------------------

**更新频率：** 周度更新。红色状态时转为日度更新。

4.5.4 人类最终裁决接口

当反身性压力计或范式稳定性监测触发到严重程度时，不自动执行决策，而是向人类呈现观察结果，请求\"最高智慧\"的介入。

**触发条件（满足任一即触发）：**

1.  反身性压力综合得分 \> 70（红色）

2.  范式稳定性判定为\"范式转移\"

3.  框架信号与 Layer 4.5 压力计方向持续冲突（连续 3 周信号方向正确但压力计维持橙色以上）

**触发后流程：**

  ----------------------- ------------------------------------------------------------------------------------------
  步骤                    动作

  1\. 自动暂停            Alpha 偏离执行力度降至 25%，新信号暂停发布

  2\. 状态呈现            向框架负责人呈现：当前信号、压力计读数、范式稳定性状态、历史类比（如有）

  3\. 定性类比            面对从未见过的冲击时，不强行量化，通过历史定性类比判断风险性质（参考案例库中的模式匹配）

  4\. 人类裁决            框架负责人做出决策：①恢复执行 ②维持观察模式 ③手动调整信号方向

  5\. 记录归档            裁决过程与结论完整记录，纳入框架迭代素材
  ----------------------- ------------------------------------------------------------------------------------------

**关键原则：** 人类裁决接口不是否定框架，而是在框架信号可靠性下降时的安全网。它确保\"最高智慧\"在关键时刻介入，而非让系统在不确定环境中自行决策。

  ----------------------------------- --------------------------------------------------------------
  修正来源                            修正系数取值

  范式稳定性（4.5.3）                 稳定 1.0 / 动摇 0.7 / 转移 0（仅维持 Beta）
  ----------------------------------- --------------------------------------------------------------

  \< 50%                  弱信号                  不产生 Alpha 偏离，仅做结构性配置

5.1 核心架构：Beta + Alpha 双层

**设计逻辑：**

- **Beta 层（All Weather 风险平价）：** 保证信号失效时仍能存活。无论宏观判断对错，Beta 层确保组合不会因单次误判遭受致命损失。这一层是\"活着\"的保障。

- **Alpha 层（预期差信号驱动）：** 让信号有效时跑出超额。当 Layer 4 的预期差信号清晰且强度足够时，Alpha 层在 Beta 基准上做方向性偏离。这一层是\"跑赢\"的引擎。

5.3 Alpha 偏离的生成

Layer 4 的预期差信号 + Layer 2.5 枢纽变量信号 → 映射为相对基准的偏离

信号映射规则

  ----------------------------- ------------------------------ -----------------------------------------------------
  信号                          方向                           Alpha 偏离

  中国增长预期差+               实际增长强于定价               A 股↑、商品↑、利率债↓

  中国通胀预期差+               实际通胀强于定价               商品↑、上游股↑、利率债↓

  中国政策预期差+               政策超预期宽松                 A 股↑、信用债↑、AI 科技链↑

  外部压力预期差−               压力小于担忧                   港股↑、外资重仓股↑、黄金↓

  中美增长预期差+               中国相对增长更被低估           A 股 vs 美股（超配 A 股）、中债 vs 美债（超配中债）

  全球紧缩三角                  Layer 2.5 三角定位为全球紧缩   整体降低风险资产、增加黄金

  铜金比极端低位 + 增长信号好   铜被过度低估 + 增长确认        周期股 vs 防御股（超配周期股，反向修复交易）

  USD/CNH 贬值压力↑             人民币贬值预期增强             港股↓、外资重仓股↓、出口股↑、黄金↑
  ----------------------------- ------------------------------ -----------------------------------------------------

**Alpha 偏离计算规则：**

1.  每个有效信号产生一个偏离方向，**信号强度取 Layer 4.5 修正后的最终强度**（而非 Layer 4 初始强度）

2.  **最终信号强度 = Layer 4 初始强度 × (1 − 累计衰减率) × 逻辑生命周期修正 × 反身性压力修正 × 范式稳定性修正**

    - 示例：初始强度 80%，B 类信号已发布 2 个月（累计衰减 40%），逻辑\"扩散期\"（×0.9），反身性\"绿色\"（×1.0），范式\"稳定\"（×1.0），最终强度 = 80% × 0.6 × 0.9 × 1.0 × 1.0 = 43.2%，低于 50% 阈值，不产生 Alpha 偏离

3.  偏离幅度 = 基准权重 × 信号强度系数（修正后强度 ≥ 80% → 系数 0.3；60%-80% → 0.2；50%-60% → 0.1；\< 50% → 不产生偏离）

4.  多个信号对同一资产产生同向偏离时，偏离幅度相加，但受 5.5 节风险预算约束上限限制

5.  多个信号对同一资产产生反向偏离时，取净值

6.  **Layer 4.5 人类裁决接口触发期间**，Alpha 偏离全部暂停，仅维持 Beta 基准

行业配置量化评分公式

**评分公式：** 行业配置得分 = 周期匹配度（0 或 1）× 3 + 景气趋势（上行 +2 / 持平 +1 / 下行 +0）+ 政策催化（有 +2 / 无 +0）

  ----------------------- --------------------------- ------------------------------------------------------------
  得分区间                配置等级                    含义

  **≥ 5 分**              **一级配置（超配）**        周期定位 + 景气指标 + 政策催化三重共振，领先受益的主流行业

  **3-4 分**              **二级配置（标配）**        周期定位受益、但景气指标或政策催化仅满足其一

  **≤ 2 分**              **三级配置（低配/规避）**   周期定位不利，或高估值、高拥挤度风险明显
  ----------------------- --------------------------- ------------------------------------------------------------

**约束规则：**

1.  单一行业评分不得仅依据\"政策催化\"单项得满分，政策催化分必须有景气数据支撑；

2.  高拥挤度行业（换手率/成交额占比过高）需在输出中标注\"拥挤度风险\"警告；

3.  行业配置结论有效期与大类资产信号有效期一致。

行业 Alpha 偏离映射规则

行业配置得分需转化为 Alpha 偏离权重，纳入 Layer 5 风险预算约束：

**映射步骤：**

1.  **行业得分 → 行业偏离方向**：得分 ≥ 5 → 超配，3-4 → 标配，≤ 2 → 低配

2.  **行业偏离幅度**：行业偏离幅度 = 宏观 Alpha 偏离幅度（5.3 节）× 行业偏离系数

    - 超配行业：偏离系数 = +0.5（即行业偏离为宏观偏离的 50%）

    - 标配行业：偏离系数 = 0（不做行业偏离）

    - 低配行业：偏离系数 = −0.5

3.  **与宏观 Alpha 叠加**：行业偏离与宏观 Alpha 偏离同向时相加，反向时取净值

4.  **纳入风险预算约束**：行业偏离后的最终权重同样受 5.5 节单资产最大权重偏离（基准的 ±50%）约束

**示例：** 宏观 Alpha 对 A 股偏离 +4.5%，\"电子\"行业得分 6 分（超配），则电子行业偏离 = 4.5% × 0.5 = +2.25%，叠加后电子配置权重在 A 股内部超配 2.25 个百分点。

4.4.2 政策超预期突变

1.  **常规政策调整：** 按原规则等待 1-2 个月核心指标验证，再按 4.2 拐点标准判定周期切换；

2.  **历史级强政策对冲（量化标准见 3.3 第 10 条）：** 立即启动高频指标跟踪，可临时调整信号方向，同步等待月度数据验证；只有满足 4.2 的拐点确认标准，才能正式判定周期切换。

------------------------------------------------------------------------

4.4.3 数据大幅修正

按修正后的数据重新执行 4.2 拐点判定，仅调整未来信号，不追溯过往结论。

4.4.4 中美周期严重分化

美联储大幅加息，国内大幅宽松（或反之），信号有效性直接下调至 **5 分及以下**，仅可输出结构性配置信号或观望信号，不做全面方向判断，等待周期收敛。

------------------------------------------------------------------------

4.4.5 流动性危机

银行间市场资金面异常紧张、美元流动性枯竭，无论周期定位如何，直接输出看空 / 观望信号，优先提示风险，待流动性环境回归正常后，再重启 4.2、4.3 常规分析。

------------------------------------------------------------------------

4.4.6 Sahm Rule 衰退预警

**触发条件：** Sahm Rule 读数 ≥ 0.5（即当月失业率 3 个月均值 - 过去 12 个月失业率最低 3 个月均值 ≥ 0.5 个百分点）

**处理规则：**

1.  触发后，无论其他指标如何，\*\*强制输出\"美国衰退预警\"\*\*标志，在报告开头置顶显示；

2.  美国基本面模块标签**强制判为\"下行\"**，不受其他指标影响；

3.  美股信号**下调 1 档**（看涨→观望，观望→看空）；

4.  A 股信号不受直接调整，但需在尾部风险提示中标注\"美国衰退风险传导\"；

5.  触发后，进入**高频跟踪模式**：每周跟踪初请失业金人数、续请失业金人数，直至 Sahm Rule 读数回落至 0.5 以下。

六、框架版本管理机制

6.1 版本号命名规则

框架采用三级版本号体系：**主版本号.次版本号**，格式为 VX.Y。

- **主版本号（X）**：当框架发生公理体系变更、核心逻辑重构或阶段映射结构重组时递增。例如从 V1.x 升级至 V2.0，表示框架底层逻辑发生根本性调整。

- **次版本号（Y）**：当框架新增指标、模块或规则，但未改变公理体系与核心阶段映射结构时递增。例如从 V1.0 升级至 V1.1，表示增量扩展。

版本号递增判定依据：

  ----------------------------------- -----------------------------------
  变更类型                            版本号变更

  新增/替换公理                       主版本号递增

  阶段映射结构重组                    主版本号递增

  核心模块新增或重构                  次版本号递增

  指标/规则增量扩展                   次版本号递增

  文字表述修正、案例更新              不递增，记录修订日期
  ----------------------------------- -----------------------------------

6.2 版本迭代触发条件

框架版本迭代由以下任一条件触发：

1.  **公理体系变更**：新增、替换或合并公理，导致底层逻辑调整。

2.  **指标体系扩展**：新增关键宏观指标或市场指标，影响信号判定逻辑。

3.  **规则量化升级**：将模糊规则量化为明确阈值，或调整既有阈值区间。

4.  **模块结构重组**：新增独立模块、合并既有模块，或调整模块间依赖关系。

5.  **回测验证驱动**：回测结果显示系统性偏差，需修正规则或参数。

6.  **重大市场事件**：出现框架未覆盖的极端市场情境，需补充规则（如黑天鹅事件分级）。

每次迭代须在变更日志中记录触发条件、变更内容及验证结论。

6.3 版本变更流程

框架版本变更遵循以下标准流程：

1.  **变更提案**：由框架维护人发起，说明变更动机、涉及模块及预期影响范围。

2.  **影响评估**：逐项评估变更对现有公理、指标、规则及信号判定的影响，确认无逻辑冲突。

3.  **草案编制**：在副本文档中完成修改，标注变更位置，确保原文可追溯。

4.  **回测验证**：对受影响模块进行回测，验证变更后信号的准确性与一致性。

5.  **审批发布**：由框架负责人审批后，更新版本号并发布正式版。

6.  **存档归档**：将前一版本全文存档，更新版本索引与变更日志。

流程关键约束：

- 变更提案须在发布前完成回测验证，禁止未经回测的规则上线。

- 涉及公理变更的提案，须经框架负责人与核心研究员双重审批。

- 每次发布后，变更日志须同步更新，确保版本可追溯。

6.4 版本存档与追溯

- **全文存档**：每个历史版本保存完整 Markdown 文件，命名格式为 macro_framework_vX.Y.md，存放于版本存档目录。

- **变更日志**：当前版本文档内保留变更日志，记录每次版本迭代的核心变更。历史版本演进以摘要形式保留，当前版本变更日志详尽记录。

- **版本索引**：维护版本索引表，包含版本号、发布日期、核心变更概述及存档文件路径。

- **可追溯性**：任何规则或指标可通过变更日志回溯至其引入版本及原始变更提案。
